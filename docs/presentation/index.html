CORE CONCEPT

三个角色，一条总线

MAIN AGENT
Team Lead

理解需求、拆解任务、组建团队、质量把关。像写 Prompt 一样为每个 Worker 编写专属角色定义和工作指令。

WORKER AGENT
全能工程师

完整的 Claude Code 实例。读写文件、执行命令、启动 subagent、使用 superpowers 技能，自主规划和交付。

BUS
通信总线

Agent 间通过 bus_send_message 直接点对点通信，无需经过 Main 中转。Unix Socket 实现，低延迟高可靠。

ARCHITECTURE

系统架构

四层架构，每层职责清晰。协议层无 IO，核心层无 UI，通信层无业务逻辑。

界面层
TUI App ← MessagesView StatusBar InputBox
调度层
Router Scheduler Channel Store
通信层
Bus Socket ↔ MCP Server ↔ AcpClient
协议层
JSON-RPC 2.0 ACP Handshake LineCodec ContentBlock
PROTOCOL

ACP 握手序列

initialize → authenticate? → session/new → session/prompt → session/update ↻

每个 Agent 通过 stdin/stdout 管道与 Bus 通信。MCP Server 内置到主二进制，通过 current_exe() 自动发现。

FEATURES

核心特性

一行创建 + 派发
创建 Agent 的同时指定任务，支持多行任务描述。Main Agent 的回复自动解析执行。
/add w1 claude 你是代码审查专家。任务：审查 PR #42...
Agent 间直接通信
Worker 通过 bus_send_message MCP 工具直接给其他 Agent 发消息，无需经过 Main 中转。
bus_send_message(to: "w2", content: "接口定义已完成，请查收")
智能调度
Main Agent 消息串行化（Scheduler 队列），防止消息风暴。Worker 之间完全并行。
@mention 路由
自动解析 @agent 提及，提取 per-agent 内容段，分发到对应 Agent。最大路由深度 5 层。
多模型混合
同一频道内混用 Claude、Gemini、Codex，按 Adapter 配置切换。每个 Agent 可以使用不同的 LLM 后端。
