# acp-bus 手动测试场景

每个场景直接复制发送给 main agent。观察 TUI 行为是否符合预期。

---

## 场景 1：群组辩论（多轮不中断）

验证：PendingTasks 不会在群组讨论中途过早触发 all-done。

```
组织一场3轮辩论赛，辩题：远程办公 vs 办公室办公。
创建3个agent：正方、反方、裁判。
创建群组，用 bus_group_message 发起讨论，设置 rounds=3。
```

**预期**：
- [ ] 3 个 agent 创建并上线
- [ ] 群组创建成功
- [ ] 辩论跑满 3 轮（每轮每人发言一次，共 9 次发言）
- [ ] main 不在中途插嘴说"辩论结束"
- [ ] 3 轮全部完成后 main 才收到系统通知并做总结

**失败标志**：main 在第 1 轮结束后就说"所有agent已完成"

---

## 场景 2：独立并行任务

验证：非群组的独立任务 agent 完成后正常触发 all-done。

```
帮我同时做两件事：
1. 调查 Rust 的 async runtime 有哪些选择
2. 调查 Go 的 goroutine 调度器原理
分别创建 agent 处理，完成后给我汇总。
```

**预期**：
- [ ] main 立即委派（不自己搜索），几秒内回复用户
- [ ] 创建 2 个 agent 各自独立工作
- [ ] 两个 agent 都完成后 main 收到通知并汇总
- [ ] main 不在只有 1 个 agent 完成时就汇总

**失败标志**：main 自己做了搜索；或只有 1 个完成就汇总

---

## 场景 3：混合场景（独立任务 + 群组讨论）

验证：独立任务和群组讨论混合时 PendingTasks 正确追踪。

```
我需要两件事：
1. 创建一个 researcher agent 去调查 SQLite 和 DuckDB 的区别
2. 同时组织一场 2 轮小辩论（2个agent讨论：嵌入式数据库 vs 客户端-服务器数据库）
全部做完后给我一个综合报告。
```

**预期**：
- [ ] 创建 3 个 agent（1 独立 + 2 辩论）
- [ ] researcher 独立完成不影响辩论进行
- [ ] 辩论跑满 2 轮
- [ ] 全部（researcher + 辩论双方）都完成后 main 才汇总

**失败标志**：researcher 完成后 main 就汇总；辩论只跑 1 轮

---

## 场景 4：委派而非自己干

验证：main 收到复杂任务后立即委派。

```
帮我全面调研一下 Tauri 这个框架，包括优缺点、和 Electron 的对比、适用场景。
```

**预期**：
- [ ] main 在 5 秒内回复用户（类似"已安排 agent 调研"）
- [ ] main 创建 agent 处理，自己不做搜索
- [ ] agent 完成后 main 汇总结果

**失败标志**：main 自己花 30+ 秒做搜索后才回复

---

## 场景 5：单轮群组消息（最小验证）

验证：rounds=1 的群组消息正常完成。

```
创建2个agent（乐观派和悲观派），建个群，让他们各说一句对AI未来的看法。一轮就够。
```

**预期**：
- [ ] 2 个 agent 各发言 1 次
- [ ] 群组讨论结束后 main 收到通知
- [ ] 不会卡死或无通知

---

## 检查点速查

| 检查项 | 命令/位置 |
|--------|----------|
| agent 是否上线 | TUI 侧栏 agent 列表，状态应为"空闲" |
| 群组消息是否完整 | 点击 TUI 侧栏群组 tab 查看 |
| main 是否过早总结 | System tab 看 "已完成" 时间戳 |
| PendingTasks 通知 | System tab 看 "[System: ..." 消息出现时机 |
| agent 任务是否送达 | agent tab 是否有消息记录 |
